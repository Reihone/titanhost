# 🐺 TitanHost Configurator

**TitanHost Configurator** — это нативное графическое приложение на Rust для управления, скачивания и настройки серверов Minecraft на Linux. Оно предоставляет удобный интерфейс (GUI) для управления игровыми ядрами, настройки сетевых параметров (включая автоматическую интеграцию с ZeroTier для игры без внешнего IP), менеджмента модов, плагинов и резервных копий.

Построен с использованием сверхбыстрой библиотеки **egui** (`eframe`) и асинхронного рантайма **Tokio**.

---

## 🚀 Основные возможности (Features)

Интерфейс приложения разделен на 4 ключевые вкладки:

1. **🎮 Запуск (Launch)**
   - **Управление профилями**: Создание, удаление и быстрое переключение между независимыми профилями серверов.
   - **Выбор версии**: Поддержка широкого диапазона версий Minecraft (от 1.7.10 до новейших).
   - **Управление памятью**: Ползунок выделения ОЗУ с автоматическим ограничением под параметры вашей системы.
   - **Умные пресеты**: Функция "Умный пресет" рассчитывает оптимальный объем памяти с учетом резервирования под ОС.
   - **Консоль логов в реальном времени**: Просмотр вывода сервера с фильтрацией логов (`Все`, `События`, `Инфо`, `Варнинги`, `Ошибки`).
   - **Отправка команд**: Возможность вводить команды (например, `op player`, `stop`, `say`) прямо в активную консоль сервера.
   - **Мониторинг ресурсов**: Отображение текущего потребления CPU и RAM системой (с использованием библиотеки `sysinfo`).
   - **Оптимизация Java**: Переключатель флагов запуска (Aikar's Flags) для оптимизации работы сборщика мусора (Garbage Collector) и минимизации фризов.

2. **🛠️ Моддинг (Modding)**
   - **Автозагрузчик ядер**:
     - **Vanilla**: Чистый официальный сервер от Mojang.
     - **Forge / NeoForge**: Скачивание и тихий запуск официальных инсталляторов.
     - **Fabric**: Автоматическая сборка сервера с Fabric Loader.
     - **Paper**: Скачивание оптимизированного ядра PaperMC (через официальное API PaperMC).
   - **Менеджер зависимостей**:
     - **Автоматическая установка Java**: Обнаружение и скачивание нужных версий Java Runtime Environment (JRE 8, 17, 21) от Adoptium (Eclipse Temurin) в локальную директорию `./run/jreX`, не засоряя системные переменные.
     - **Загрузка библиотек**: Скачивание необходимых клиентских библиотек Minecraft.
   - **Менеджер модов и плагинов**: Сканирование локальной директории, просмотр списка модов (`.jar`) и плагинов, синхронизация.

3. **📡 Сетевые настройки (Server)**
   - **Редактор `server.properties`**: Настройка основных параметров сервера прямо из GUI (проверка лицензии `online-mode`, PvP, белый список, полеты, сложность, спавн мобов, количество игроков, дальность прорисовки и т.д.).
   - **Мониторинг сети (Pinger)**: Встроенный пингер для проверки доступности указанного IP и порта.
   - **Интеграция с ZeroTier**:
     - Получение статуса локального демона ZeroTier One.
     - Автоматический безопасный запрос авторизационного токена (`authtoken.secret`) через PolicyKit (`pkexec`), с последующим кэшированием в `~/.config/titanhost/zerotier_token.txt`.
     - Присоединение (Join) и выход (Leave) из виртуальных сетей прямо из интерфейса для быстрого соединения с друзьями без port forwarding.

4. **💾 Резервные копии (Backups)**
   - Создание полных бэкапов мира и конфигурации в формате `.zip`.
   - Просмотр списка бэкапов с указанием времени создания и размера файлов.
   - Восстановление бэкапа в один клик с автоматической заменой файлов мира.
   - Удаление старых бэкапов.

## 🐧 Поддержка дистрибутивов Linux (Distros Support)

Поскольку приложение скомпилировано на Rust и использует OpenGL (через `glutin`/`wgpu` в составе `eframe`), оно совместимо с большинством современных дистрибутивов Linux.

| Семейство дистрибутивов | Поддержка | Требуемые системные зависимости |
| :--- | :--- | :--- |
| **Ubuntu / Debian / Mint / Pop!_OS** | Полная | `libxcb-render0-dev libxcb-shape0-dev libxcb-xfixes0-dev libx11-xcb-dev libasound2-dev libudev-dev` |
| **Arch Linux / Manjaro** | Полная | Устанавливается из коробки, для компиляции: `base-devel`, `alsa-lib`, `systemd-libs` |
| **Fedora / RedHat** | Полная | `alsa-lib-devel systemd-devel libX11-devel libXcursor-devel libXi-devel libXrandr-devel libXinerama-devel libglvnd-devel` |
| **Gentoo / Void / Alpine** | Частичная | Требуется наличие glibc (на Alpine с musl может потребоваться статическая сборка или gcompat) |

*Примечание: Поддерживаются оконные менеджеры как на базе **X11**, так и **Wayland**.*

## 🛠 Технологический стек (Under the Hood)

В проекте используются современные, высокоэффективные библиотеки экосистемы Rust:

- **Язык**: Rust (Edition 2021) — гарантирует безопасность работы с памятью и максимальную производительность без сборщика мусора.
- **Интерфейс (GUI)**: [egui (eframe)](https://github.com/emilk/egui) версии `0.25.0` — сверхлегкая Immediate-mode графическая библиотека.
- **Асинхронность**: [Tokio](https://tokio.rs/) — многопоточный рантайм для фоновых скачиваний и запуска процессов сервера.
- **Сетевые запросы**: [reqwest](https://github.com/seanmonstar/reqwest) — загрузка ядер, манифестов и JRE.
- **Системные показатели**: [sysinfo](https://github.com/GuillaumeGomez/sysinfo) — мониторинг нагрузки на процессор и занятой оперативной памяти.
- **Интеграция с ZeroTier**: Использование локального API демона ZeroTier One (`http://127.0.0.1:9993`) с авторизацией через заголовки `X-ZT1-Auth`.
- **Сериализация данных**: `serde` и `serde_json` для конфигурационных файлов (`profiles.json`, `server.properties`).
- **Сжатие данных**: `zip` для резервных копий.

## 📦 Сборка и Установка (Build & Install)

### 1. Установка системных зависимостей

Перед компиляцией убедитесь, что в вашей системе установлены графические библиотеки разработки.

**Для Ubuntu / Debian / Pop!_OS:**
```bash
sudo apt-get update
sudo apt-get install -y build-essential libxcb-render0-dev libxcb-shape0-dev libxcb-xfixes0-dev libx11-xcb-dev libglu1-mesa-dev libasound2-dev libudev-dev libwayland-dev
```

**Для Arch Linux / Manjaro:**
```bash
sudo pacman -Syy --needed base-devel alsa-lib libx11 libxcursor libxinerama libxrandr libxi libglvnd
```

**Для Fedora:**
```bash
sudo dnf groupinstall "Development Tools"
sudo dnf install alsa-lib-devel systemd-devel libX11-devel libXcursor-devel libXi-devel libXrandr-devel libXinerama-devel libglvnd-devel
```

### 2. Сборка из исходников

Для сборки проекта вам понадобится установленный инструментарий Rust ([Rustup](https://rustup.rs/)):

```bash
# Клонирование репозитория
git clone https://github.com/username/titanhost.git
cd titanhost

# Сборка оптимизированного релизного бинарника
cargo build --release
```

После завершения компиляции исполняемый файл будет находиться по пути:
`./target/release/titanhost`

### Важное при выкладывании на GitHub:
Папка `/run/` добавлена в `.gitignore`. Она содержит все локально скачанные бинарные файлы серверов (`.jar`), встроенные Java JRE, логи серверов, миры и плагины. Это сделано специально, чтобы не загружать гигабайты временных и пользовательских данных в репозиторий. При запуске собранного приложения на чистом Linux все необходимые папки и базовые конфиги (`profiles.json`) создаются автоматически в директории `./run/` при первом запуске.

## 🔒 Безопасность и ZeroTier

Для получения токена авторизации ZeroTier приложение использует утилиту `pkexec` для чтения файла `/var/lib/zerotier-one/authtoken.secret` с правами суперпользователя. При этом:
1. Система запрашивает графический или консольный пароль администратора (через PolicyKit).
2. Полученный токен кэшируется в файл `~/.config/titanhost/zerotier_token.txt` с правами текущего пользователя.
3. В последующие разы приложение считывает токен из кэша, не требуя повторного ввода пароля.
4. Приложение общается исключительно с локальным демоном ZeroTier (`127.0.0.1:9993`) и никогда не передает ваши ключи во внешнюю сеть.
